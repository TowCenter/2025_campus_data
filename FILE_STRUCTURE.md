## File Purposes

### Core Application Files

**`App.svelte`** - Main application
- Handles page routing (Methodology â†” Data)
- Renders header with navigation
- Renders footer
- Wraps page content

**`Methodology.svelte`** - Methodology page
- Displays methodology text (editable placeholders)
- Has sections for embedding charts
- Styled for readability

**`DataExplorer.svelte`** - Data & downloads page
- Fetches data from AWS S3
- Displays searchable table of universities
- Handles checkbox selection
- Generates CSV and JSON downloads
- Excludes `llm_response` and `scraper` from exports
- Shows "Last Updated" timestamp

### Configuration Files

**`package.json`**
- Project metadata
- Dependencies (Svelte, Vite)
- NPM scripts (dev, build, preview)

**`vite.config.js`**
- Vite bundler configuration
- Development server settings
- Build output settings

**`svelte.config.js`**
- Svelte compiler settings
- Preprocessing configuration

**`index.html`**
- HTML shell
- Loads the Svelte app
- Meta tags for SEO

**`main.js`**
- Entry point
- Instantiates the Svelte app

## What to Edit

### âœï¸ You WILL edit these:

1. **`DataExplorer.svelte`** (line 5)
   - Update S3_BUCKET_URL with your AWS URL

2. **`Methodology.svelte`**
   - Replace placeholder text with your methodology
   - Add your actual charts

3. **`package.json`** (optional)
   - Update name, description, author

### âš ï¸ You probably WON'T edit these:

- `App.svelte` - Unless changing navigation
- `main.js` - Entry point, rarely needs changes
- `index.html` - Might update title/meta tags
- `vite.config.js` - Works as-is
- `svelte.config.js` - Works as-is

### ğŸš« You should NOT edit these:

- `node_modules/` - Managed by npm
- `package-lock.json` - Managed by npm
- `dist/` - Generated by build

## Adding New Features

### Add a new page:

1. Create `NewPage.svelte` component
2. Import in `App.svelte`
3. Add to navigation
4. Add routing logic

### Add a chart library:

```bash
npm install chart.js
# or
npm install recharts
# or
npm install d3
```

Then import in your component:
```javascript
import Chart from 'chart.js/auto';
```

### Add styling:

Styles are component-scoped in `<style>` tags within each `.svelte` file.

## Development Workflow

```bash

# 2. See changes instantly

# 3. Test in browser
http://localhost:3000

# 4. Build for production

# 5. Preview production build

# 6. Deploy
# (Push to GitHub or upload dist/ folder)
```
